三个结论：

1. 旧论文的 ID/OOD 方向在新模型上反转：三个模型都是 ID 上 static≥dynamic、

OOD 上 dynamic≥static（Haiku OOD 差 18 分）。"低 coverage → 用 generic" 的处方已失效。

2. Per-query 异质性在中档模型上依然巨大：Sonnet 4.6 的 oracle 比最优单方法高

+14（ID）/+8（OOD），Haiku +14/+4——per-query routing 这个 meta-idea 有真实 headroom，

但 routing 信号需要重新学/设计，不能再用 "低 coverage→static" 的固定方向。

3. 最强模型饱和：gemini-3-flash ID 98-100、分歧 1/50——对 frontier 模型

selection 已几乎不重要；价值在 cheap/mid 模型（正是部署中省钱的场景）。

另：gemini-2.0-flash 与 gecko embedding API 已下线，主表无法在原模型上复跑。

含义（待与 Colin 讨论）：renovate（把旧 claim 搬到新模型）不可行；

建议 pivot scope → "per-query context/compute routing for cost-efficient reasoning"：

用 zero-cost 信号预测每个 query 该给哪种 context（或哪个模型/多少算力），

oracle gap 就是 motivation，COLM workshop 版可以先以 test-free + efficiency 故事投出去

（其结果基于已完成的 2.0-flash runs，如实标注），大 refactor 走新 scope。
