Merged conclusions (with the API-model results below):

1. The old "dynamic wins ID / static wins OOD" rule holds on none of 7 models; OOD collapses to 0 below 70B (unsolvable, not static-favoring).

2. Oracle headroom emerges with capability and peaks in the mid-band (+16 Qwen-72B / +14 Llama-70B / +14 Sonnet / +14 Haiku; ≈0 for Gemini 3 Flash). The ID winner is family-dependent (Qwen leans static, Llama dynamic) → fixed rules are dead; routers must be model-aware.

3. Decoding does not flip the selection winner; SC-majority ≤ greedy; but 32B static pass@10 = 48 vs greedy 38 → the diversity exists and majority voting cannot cash it — validator-guided best-of-n can (we have a free verifier).

1. The original paper's ID/OOD direction is reversed on all three models (static ≥ dynamic on ID; dynamic ≥ static on OOD, by 18 points on Haiku). The "low coverage → generic exemplars" prescription no longer holds.

2. Per-query heterogeneity remains large on mid-tier models (oracle +8 to +14) — the routing meta-idea has real headroom, but the routing signal must be redesigned.

3. Frontier models saturate the task (Gemini 3 Flash: 98–100 on ID, 1/50 disagreement) — selection barely matters at the frontier; the value concentrates in the cheap/mid deployment tier.

Also: gemini-2.0-flash and the gecko embedding API were shut down (404) — the original main-table model can no longer be re-run; all new runs use the Bedrock/Vertex backends.
